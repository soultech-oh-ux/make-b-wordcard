# data/ — 성경 구절 데이터

말씀카드 메이커가 페이지 로드 시 이 폴더에서 JSON을 불러와 "구절 선택" 드롭다운, "직접 입력" 변환, "본문 자동 채우기"에 사용합니다.

## 파일

| 파일 | 내용 | 출처/저작권 |
|---|---|---|
| `bible_books.json` | 성경 66권 매핑 (한글·영문·약칭) | 책 이름은 사실 정보 |
| `verses.json` | 추천 구절 **400개** (20주제 × 20구절) — 참조 메타만 | 참조만 |
| `texts_kjv.json` | KJV 영문 본문 (스크립트로 채움) | **퍼블릭 도메인** |
| `texts_web.json` | WEB 영문 본문 (스크립트로 채움) | **퍼블릭 도메인** |

## 본문은 어떻게 채우나

처음에는 `texts_kjv.json` / `texts_web.json`이 비어 있습니다(`{}`). Node 스크립트를 한 번 실행하면 bible-api.com에서 정확한 본문을 받아 채웁니다:

```bash
# Node.js 18 이상 필요
node scripts/fetch_verse_texts.js
# 한 쪽만 받기
node scripts/fetch_verse_texts.js --only=kjv
node scripts/fetch_verse_texts.js --only=web
```

- 이미 받은 키는 건너뛰므로(증분 실행) 중간에 멈춰도 다음에 이어갑니다.
- 50개마다 자동 저장됩니다.
- 요청 사이 120ms 지연이 들어 있어 API에 부담을 주지 않습니다.

채워두면 앱이 KJV/WEB 선택 시 **로컬 본문을 먼저 사용**(오프라인에서도 즉시) 하고, 없는 항목만 API를 호출합니다.

## 자유 입력은 어떻게 동작하나

사용자가 "구절 직접 입력" 칸에 다음 형식 중 하나로 입력 가능:

```
창세기 1:1
마 5:3-10
요한복음 3:16
1 corinthians 13:4-7
```

앱이 `bible_books.json`의 `ko`/`aliases`/`en`을 사용해 책을 찾고, 장:절 범위를 파싱해서 기존 `fetchVerse` 로직을 그대로 호출합니다.

## 추천 구절 추가하기

`verses.json`의 `verses` 배열에 아래 형식으로 항목을 추가하세요. 페이지를 새로고침하면 자동으로 드롭다운에 반영됩니다.

```json
{ "label": "창세기 1:1", "category": "찬양", "bk": 1, "ch": 1, "v1": 1, "v2": 1, "en": "genesis 1:1" }
```

카테고리를 새로 만들고 싶으면 `categories` 배열에도 함께 추가하세요.

## 본문(text)을 직접 담아도 되나요?

- **퍼블릭 도메인 번역(KJV, WEB)**: 안전합니다 — 위 스크립트가 자동으로 `texts_kjv.json` / `texts_web.json`에 저장합니다.
- **개역한글(1961)**: 한국 저작권 상태가 회색지대라 이 저장소에는 담지 않습니다. 앱은 런타임에 외부 API(getbible.net)에서 가져옵니다.
- **개역개정 / NIV / 쉬운성경 등 저작권 보호 번역**: 본문을 파일에 담거나 재배포하면 **저작권 침해**가 됩니다. 앱은 해당 번역의 경우 참조만 자동 입력하고, 사용자가 본인이 보유한 성경에서 본문을 직접 붙여넣도록 안내합니다.
- **개인용 자료실(앱 안 ⑤ 내 구절 모음)**: localStorage에만 저장되어 GitHub/Vercel에 올라가지 않으므로, 본인이 가진 책에서 옮긴 본문을 개인 사용 범위로 자유롭게 사용할 수 있습니다.

## 오프라인 동작

- 사용자가 `index.html`을 `file://` 로 직접 열면 fetch가 차단될 수 있습니다. 이 경우 앱은 내장된 fallback 데이터(코드 안의 작은 기본 목록)를 사용합니다.
- Vercel 등으로 https 배포하면 이 폴더의 JSON이 정상적으로 로드되고 PWA로 캐시됩니다(서비스워커 `sw.js`).
